# Environment Setup

This project uses environment variables for API keys and sensitive configuration.

## Quick Start

1. Copy the example environment file:
   ```bash
   cp .env.example .env
   ```

2. Edit `.env` and add your API keys:
   - Get your Open Router API key from: https://openrouter.ai/keys
   - Add other API keys as needed

3. The `.env` file is automatically ignored by git (see `.gitignore`)

## Environment Variables

- `OPENROUTER_API_KEY`: Required for Open Router models (e.g., Llama 3.3 70B)
- `OPENAI_API_KEY`: Required if using OpenAI models (GPT-4, etc.)

## Loading Environment Variables

The project uses `python-dotenv` to automatically load variables from `.env`:

```python
from dotenv import load_dotenv
load_dotenv()  # Load environment variables from .env file
```

This is typically done at the application entry point before importing any modules that need the API keys.
